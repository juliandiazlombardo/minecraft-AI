# Agent Logger and Knowledge Base System Requirements

I need you to create a simple, reusable logging and tracking system for my Python-based AI agent project. The primary goal is for the AI agents running in the project to seamlessly use this system to guide their actions, maintain a history of their runs, and leverage a knowledge base of past errors and solutions, while keeping the main project directory perfectly clean.

Here are the specific requirements:

## Core Objective
Build a dual-purpose system:
1.  **Run Recordings:** Automatically record system runs and errors in a combination of two formats: JSON/JSONL (structured data for the AI agent to easily parse programmatically) and a Markdown summary file (for easy human readability).
2.  **Knowledge Base Registry:** Create an automated Markdown registry (e.g., `troubleshooting_registry.md`) structured as a simple table: `| Error Symptom | Root Cause | Solution Steps | Certainty Score |`.

## Important Context
- **The Certainty Score:** The registry must use a `certainty-of-fix` score from 0 to 2 to evaluate solutions:
  - **Score 0:** All unfixed bugs, or bugs that have reappeared after a previous fix attempt.
  - **Score 1:** The agent believes the bug has been successfully fixed based on its run context.
  - **Score 2:** Both the human and the agent have confirmed the bug is fixed.
- **Agent Interaction:** The agents must autonomously *read* from the registry when they encounter a new error, and they must automatically *write/update* the table (updating the certainty score) when they believe they have resolved an error.
- **Standard Libraries Only:** To keep it simple and portable, you must *strictly* use only standard Python built-in libraries (such as `logging`, `json`, `os`, `pathlib`). Do not use any external dependencies.
- **Standardized Pathing (The _agents framework):** All Python scripts related to this system must live in `_agents/scripts/agent_logger.py`. All output logs and registries generated by this system must be dynamically scoped to the name of the file being executed (e.g. `__file__`) and saved inside a dedicated `_agents/logs/<source_name>/` directory. This ensures the main project root remains untouched and the entire AI history is isolated and portable. Using `_agents` (with underscore) instead of `.agents` (with dot) allows Python to import from it natively without any `importlib` hacks.
- **Portable Setup:** The implementation must include a `run_with_logging(source_file, main_func)` convenience function that wraps any main function with automatic logging, error catching (including `BaseException` like `SystemExit`), and traceback recording. Initializing the logger in any file should be as simple as:
  ```python
  if __name__ == "__main__":
      from _agents.scripts.agent_logger import run_with_logging
      run_with_logging(__file__, main)
  ```
- **Workflow/Skill Integration:** I need instructions and a clean structure so this logging system can be easily implemented and provided to an agent as a "Workflow" (e.g. `_agents/workflows/agent_logger.md`). The agent using the skill/workflow should immediately understand how to import the logger, how to log to the system, how to query it when stuck, and how to append to the registry when a solution is found/score updated.

## Things to Avoid
- Avoid overly complex database schemas (no SQL, no heavy vector DBs). Stick exclusively to simple local file operations within `_agents/logs/`.
- Avoid polluting the global project namespace with logs; always scope logs relative to the script running them.

## Expected Output
1. The Python code for the Logger/Registry manager implemented as `_agents/scripts/agent_logger.py`.
2. An example of the 2-line boilerplate needed to initialize it in any script's `if __name__ == "__main__":` block.
3. An outline of the Workflow `.md`/Skill `.md` file that teaches the agent exactly how to interact with this new system.
